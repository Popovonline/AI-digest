# Calibration Log

Track scoring feedback to improve accuracy over time.

## How to Use

After each digest, review items and provide feedback:
- **Too high:** Item was rated higher than it deserved
- **Too low:** Item was rated lower than it deserved
- **Correct:** Rating was accurate

## Feedback Format

```markdown
### YYYY-MM-DD

**Item:** [Article title]
**Scored:** ★★★★☆ (I:4 R:4 C:4 = 64)
**Feedback:** too high / too low / correct
**Reason:** [Why the rating was wrong]
**Adjustment:** [What to change in scoring]
```

## Example Entry

### 2026-02-06

**Item:** GitHub Actions critique
**Scored:** ★★★☆☆ (I:3 R:3 C:3 = 27)
**Feedback:** correct
**Reason:** Opinion piece, not actionable for leadership
**Adjustment:** None needed
